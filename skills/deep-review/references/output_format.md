# Output Format

The final review must be markdown and should be direct, evidence-heavy, and calibrated.

## Required Sections

```markdown
## Overall assessment

2-3 sentences. State what the artifact tries to do, whether the evidence supports it, the biggest obstacle, and confidence.

## Critical issues

### 1. Finding title

- **Location:** Exact section, page, equation, table, figure, file, line, or passage.
- **Problem:** What is wrong.
- **Why it matters:** How it affects the claim.
- **Suggested fix:** What would repair, weaken, or clarify the claim.
- **Confidence:** High / medium / low.

## Significant concerns

Same finding format.

## Minor issues

Concise bullets are acceptable when issues are local.

## Questions for the authors

Questions that might resolve apparent problems.
```

## Finding Rules

- Do not include praise unless it helps calibrate a concern.
- Do not list a finding unless it survived self-critique.
- Prefer exact citations over broad references.
- If evidence is uncertain, say so.
- If no critical issues were found, say that clearly and list residual review limits.
- When a computational check was performed, include the method and result briefly.
